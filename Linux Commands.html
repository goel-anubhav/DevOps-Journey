Comprehensive Linux Commands Guide for DevOps Engineers

This guide includes essential Linux commands for DevOps engineers, with examples and separate copy buttons for commands and examples.

1. File and Directory Management

1.1 `pwd` - Print Current Directory

Command:
pwd


Example:
pwd
# Output: /home/user

1.2 `ls` - List Directory Contents

Command:
ls -l


Example:
ls -l
# Output: Detailed list of files and directories.

1.3 `mkdir` - Create Directory

Command:
mkdir new_folder


Example:
mkdir new_folder
# Creates a directory named new_folder.

2. User and Group Management

2.1 `adduser` - Add New User

Command:
sudo adduser username


Example:
sudo adduser devops_user
# Adds a new user named devops_user.

2.2 `usermod` - Modify User

Command:
sudo usermod -aG groupname username


Example:
sudo usermod -aG sudo devops_user
# Adds the user devops_user to the sudo group.

3. Process Management

3.1 `ps` - Show Running Processes

Command:
ps aux


Example:
ps aux | grep nginx
# Lists all running processes and filters for nginx.

3.2 `kill` - Terminate Process

Command:
kill -9 PID


Example:
kill -9 12345
# Terminates the process with PID 12345.

4. Networking

4.1 `ping` - Test Connectivity

Command:
ping google.com


Example:
ping -c 4 google.com
# Sends 4 ICMP packets to google.com.

4.2 `scp` - Secure File Copy

Command:
scp file.txt user@remote_host:/path/to/destination


Example:
scp backup.tar.gz devops@192.168.1.100:/home/devops/backups/
# Copies backup.tar.gz to the backups directory of the remote server.
